# Web应用开发课程综合实验 - 课程设计文档

## 一、项目概述

### 1.1 项目名称
校园公告信息管理系统

### 1.2 选题
**选题一：多标签信息管理系统**

### 1.3 项目简介
本系统是一个基于Spring Boot + MyBatis + Thymeleaf开发的校园公告信息管理系统，实现了公告的发布、浏览、分类、标签管理、收藏等功能。系统分为用户端（前台）和管理端（后台）两部分，满足课程设计的所有要求。

### 1.4 开发环境
- **操作系统：** Windows 10/11
- **开发工具：** IntelliJ IDEA / Eclipse
- **JDK版本：** JDK 1.8
- **数据库：** MySQL 8.0
- **构建工具：** Maven 3.6+
- **浏览器：** Chrome / Edge

## 二、需求分析

### 2.1 功能需求

#### 2.1.1 用户端功能
1. **用户注册与登录**
   - 用户注册：填写用户名、密码、昵称、邮箱等信息
   - 用户登录：使用用户名和密码登录，需要验证码验证
   - 安全退出：清除Session，退出登录状态

2. **公告浏览**
   - 浏览所有已发布的公告
   - 按分类筛选公告
   - 按关键词搜索公告
   - 查看置顶公告
   - 查看公告详情，自动增加浏览次数

3. **收藏功能**
   - 收藏感兴趣的公告
   - 取消收藏
   - 查看"我的收藏"列表

4. **个人资料管理**
   - 修改昵称、邮箱、手机号
   - 上传和更换头像

#### 2.1.2 管理端功能
1. **管理员登录**
   - 使用管理员账号登录后台
   - 权限验证

2. **公告管理**
   - 新增公告：填写标题、内容、选择分类、标签，上传封面图片
   - 修改公告：编辑公告信息
   - 删除公告：删除不需要的公告
   - 查询公告：查看所有公告列表
   - 设置置顶：将重要公告置顶显示

3. **分类管理**
   - 新增分类：创建新的公告分类
   - 修改分类：编辑分类名称、描述、排序
   - 删除分类：删除不需要的分类（需检查是否有公告使用）
   - 查询分类：查看所有分类列表

4. **标签管理**
   - 新增标签：创建新的公告标签，设置颜色
   - 修改标签：编辑标签名称和颜色
   - 删除标签：删除不需要的标签
   - 查询标签：查看所有标签列表

5. **公告与标签关联**
   - 为公告添加多个标签
   - 修改公告的标签关联

6. **数据统计**
   - 按分类统计公告数量
   - 按标签统计公告数量
   - 统计用户总数、公告总数等

7. **操作日志**
   - 记录管理员的新增、修改、删除操作
   - 记录操作时间、IP地址等信息
   - 查看最近的操作日志

### 2.2 非功能需求
1. **性能要求**
   - 页面响应时间 < 2秒
   - 支持并发访问

2. **安全性要求**
   - 登录验证码防止暴力破解
   - Session会话管理
   - 拦截器实现权限控制

3. **易用性要求**
   - 界面美观、操作简单
   - 响应式设计，支持不同屏幕尺寸

## 三、系统设计

### 3.1 系统架构
本系统采用经典的三层架构设计：

```
┌─────────────────────────────────────┐
│        表现层 (Presentation)         │
│   Thymeleaf + Bootstrap + jQuery    │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         业务层 (Business)            │
│      Controller + Service           │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│        持久层 (Persistence)          │
│        MyBatis + MySQL              │
└─────────────────────────────────────┘
```

**架构说明：**
- **表现层：** 使用Thymeleaf模板引擎渲染HTML页面，Bootstrap提供UI组件，jQuery处理前端交互
- **业务层：** Controller接收请求并调用Service处理业务逻辑
- **持久层：** MyBatis操作MySQL数据库，完成数据的增删改查

#### 4.1.1 Spring Boot配置
使用`application.yml`进行项目配置：

#### 4.1.2 MyBatis实现
采用注解方式开发，主要注解：
- `@Select`：查询操作
- `@Insert`：插入操作
- `@Update`：更新操作
- `@Delete`：删除操作
- `@Results`：结果映射

示例代码：
```java
@Select("SELECT * FROM notice WHERE id = #{id}")
Notice findById(Long id);

@Insert("INSERT INTO notice(title, content, category_id, author_id) " +
        "VALUES(#{title}, #{content}, #{categoryId}, #{authorId})")
@Options(useGeneratedKeys = true, keyProperty = "id")
int insert(Notice notice);
```

#### 4.1.3 RESTful API设计
使用标准的HTTP方法：
- `GET`：查询资源
- `POST`：创建资源
- `PUT/PATCH`：更新资源
- `DELETE`：删除资源

示例：
```java
@GetMapping("/notice/{id}")          // 查询公告详情
@PostMapping("/admin/notice/save")   // 保存公告
@DeleteMapping("/admin/notice/{id}") // 删除公告
```

#### 4.1.4 拦截器实现
实现`HandlerInterceptor`接口，在`preHandle`方法中进行权限验证：

```java
public class LoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) throws Exception {
        User user = (User) request.getSession().getAttribute("user");
        if (user == null) {
            response.sendRedirect("/user/login");
            return false;
        }
        return true;
    }
}
```

#### 4.1.5 文件上传实现
使用`MultipartFile`接收上传的文件：

```java
@PostMapping("/uploadAvatar")

    String avatarPath = "/uploads/avatars/" + fileName;

}
```

#### 4.1.6 日志管理

### 4.2 核心代码展示

#### 4.2.1 用户登录（UserController.java）
```java
@PostMapping("/login")
@ResponseBody
public Result<User> login(@RequestParam String username,
                          @RequestParam String password,
                          @RequestParam String captcha,
                          HttpSession session) {
    // 验证验证码
    String sessionCaptcha = (String) session.getAttribute("captcha");
    if (sessionCaptcha == null || !sessionCaptcha.equalsIgnoreCase(captcha)) {
        return Result.error("验证码错误");
    }

    // 验证用户名和密码
    User user = userService.login(username, password);
    if (user == null) {
        return Result.error("用户名或密码错误");
    }

    // 保存到Session
    session.setAttribute("user", user);
    return Result.success("登录成功", user);
}
```

#### 4.2.2 公告查询（NoticeService.java）
```java
public List<Notice> findAll(Long categoryId, String keyword) {
    List<Notice> notices = noticeMapper.findAll(categoryId, keyword);
    // 加载每个公告的标签
    for (Notice notice : notices) {
        notice.setTags(tagMapper.findByNoticeId(notice.getId()));
    }
    return notices;
}
```

#### 4.2.3 操作日志记录（OperationLogService.java）
```java
public void log(String operation, String method, String params, 
                HttpServletRequest request) {
    OperationLog log = new OperationLog();
    log.setOperation(operation);
    log.setMethod(method);
    log.setParams(params);
    log.setIp(getIpAddress(request));

    // 从session获取用户信息
    User user = (User) request.getSession().getAttribute("user");
    if (user != null) {
        log.setUserId(user.getId());
        log.setUsername(user.getUsername());
    }

    operationLogMapper.insert(log);
    logger.info("操作日志: {} - {} - {}", 
                log.getUsername(), operation, method);
}
```

### 7.3 遇到的问题及解决方案

**问题1：文件上传路径问题**
- 问题描述：上传的文件无法访问
- 解决方案：在WebConfig中配置资源映射，将虚拟路径映射到实际文件路径

**问题2：MyBatis关联查询**
- 问题描述：公告需要同时查询分类名称和标签列表
- 解决方案：使用多次查询组装数据，在Service层加载关联数据

**问题3：Session管理**
- 问题描述：用户信息更新后Session未同步
- 解决方案：每次更新后重新从数据库查询并更新Session
