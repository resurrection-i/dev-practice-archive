## 第七部分：Q&A环节（3-5分钟）

### 常见问题准备

**Q1：为什么选择这个题目？**
A：校园公告管理是学校信息化建设的重要组成部分，该题目功能明确、技术全面，能够充分展示所学知识。

**Q2：系统的核心技术是什么？**
A：核心技术是Spring Boot + MyBatis + Thymeleaf，采用三层架构设计，使用RESTful风格API，拦截器实现权限控制。

**Q3：如何保证系统的安全性？**
A：
1. 登录验证码防止暴力破解
2. Session会话管理
3. 拦截器实现权限控制
4. 操作日志记录
5. SQL注入防护（MyBatis参数化查询）

**Q4：如何实现公告与标签的多对多关系？**
A：使用中间表notice_tag存储公告和标签的关联关系，在Service层组装数据。

**Q5：文件上传是如何实现的？**
A：使用MultipartFile接收上传文件，生成唯一文件名，保存到指定目录，返回访问路径。

**Q6：操作日志是如何记录的？**
A：在Controller层调用OperationLogService记录操作，包括操作类型、方法、参数、IP地址等信息。

**Q7：如果公告数量很多，如何优化性能？**
A：可以实现分页功能、添加索引、使用Redis缓存热门公告等。

**Q8：系统有哪些不足之处？**
A：
1. 密码明文存储，应该加密
2. 没有实现分页功能
3. 没有使用富文本编辑器
4. 没有图片压缩功能
