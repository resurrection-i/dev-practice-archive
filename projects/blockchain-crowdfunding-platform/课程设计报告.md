# 基于区块链的众筹平台课程设计报告

## 一、项目概述

### 1.1 项目背景

传统众筹平台存在以下问题：
- **信任问题**：资金流向不透明，捐款者无法确认资金是否被正确使用
- **中心化风险**：平台可能挪用资金或跑路
- **高额手续费**：中介平台收取高额服务费
- **资金安全**：中心化存储存在被攻击风险

区块链技术的特性可以有效解决这些问题：
- **去中心化**：无需信任中介，智能合约自动执行
- **透明性**：所有交易记录公开可查
- **不可篡改**：数据一旦上链无法修改
- **低成本**：减少中介费用

### 1.2 项目目标

开发一个基于区块链的去中心化众筹平台，实现：
1. 项目创建者可以发起众筹项目
2. 支持者可以向项目捐款
3. 创建者可以创建资金使用请求
4. 支持者投票批准资金使用
5. 所有交易记录透明可查
6. 用户身份认证和钱包绑定

## 三、核心功能实现

### 3.1 智能合约实现

#### 3.1.1 项目创建
```solidity
function createProject(
    string memory _name,
    string memory _description,
    uint256 _goal,
    uint256 _duration
) public {
    require(_goal > 0, "Goal must be greater than 0");
    require(_duration > 0, "Duration must be greater than 0");
    
    Project memory newProject = Project({
        creator: payable(msg.sender),
        name: _name,
        description: _description,
        goal: _goal,
        deadline: block.timestamp + _duration,
        currentAmount: 0,
        contributorsCount: 0,
        completed: false
    });
    
    projects.push(newProject);
    emit ProjectCreated(projects.length - 1, msg.sender, _name, _goal);
}
```

#### 3.1.2 捐款功能
```solidity
function fundProject(uint256 _projectId) public payable {
    require(_projectId < projects.length, "Invalid project ID");
    require(msg.value > 0, "Contribution must be greater than 0");
    
    Project storage project = projects[_projectId];
    require(block.timestamp < project.deadline, "Project deadline passed");
    require(!project.completed, "Project already completed");
    
    // 记录贡献
    if (contributions[_projectId][msg.sender] == 0) {
        contributors[_projectId].push(msg.sender);
        project.contributorsCount++;
    }
    
    contributions[_projectId][msg.sender] += msg.value;
    project.currentAmount += msg.value;
    
    emit ProjectFunded(_projectId, msg.sender, msg.value);
}
```

#### 3.1.3 资金请求投票
```solidity
function approveRequest(uint256 _projectId, uint256 _requestId) public {
    require(_projectId < projects.length, "Invalid project ID");
    Project storage project = projects[_projectId];
    
    require(contributions[_projectId][msg.sender] > 0, "Not a contributor");
    require(_requestId < fundingRequests[_projectId].length, "Invalid request ID");
    
    FundingRequest storage request = fundingRequests[_projectId][_requestId];
    require(!request.completed, "Request already completed");
    require(!approvals[_projectId][_requestId][msg.sender], "Already approved");
    
    approvals[_projectId][_requestId][msg.sender] = true;
    request.approvalCount++;
    
    emit RequestApproved(_projectId, _requestId, msg.sender);
}
```

```java
public Result<String> register(@RequestBody RegisterRequest request) {
    // 检查用户名是否存在
    if (userService.existsByUsername(request.getUsername())) {
        return Result.error("用户名已存在");
    }
    
    // 加密密码
    String encodedPassword = passwordEncoder.encode(request.getPassword());

    user.setUsername(request.getUsername());
    user.setPassword(encodedPassword);
    user.setEmail(request.getEmail());
    
    userService.save(user);
    return Result.success("注册成功");
}
```

#### 3.3.2 用户登录

```java
@PostMapping("/login")
public Result<LoginResponse> login(@RequestBody LoginRequest request) {
    // 验证用户名密码
    User user = userService.getByUsername(request.getUsername());
    if (user == null || !passwordEncoder.matches(request.getPassword(), user.getPassword())) {
        return Result.error("用户名或密码错误");
    }
    
    // 生成JWT token
    String token = jwtUtil.generateToken(user.getId(), user.getUsername());
    
    LoginResponse response = new LoginResponse();
    response.setToken(token);
    response.setUserId(user.getId());
    response.setUsername(user.getUsername());
    
    return Result.success(response);
}
```

```javascript
请切换到正确的钱包地址，或使用正确的账户登录。`);
```

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) {
        String token = extractToken(request);
        if (token != null && jwtUtil.validateToken(token)) {
            Long userId = jwtUtil.getUserIdFromToken(token);
            // 设置认证信息
            Authentication auth = new UsernamePasswordAuthenticationToken(
                userId, null, Collections.emptyList());
            SecurityContextHolder.getContext().setAuthentication(auth);
        }
        filterChain.doFilter(request, response);
    }
}
```

## 八、遇到的问题及解决方案

### 8.1 问题1：ethers.js v6 异步问题

**问题描述**：
ethers.js v6中`provider.getSigner()`是异步方法，但代码中同步调用导致错误。

**解决方案**：
```javascript
// 错误写法
const signer = provider.getSigner();

// 正确写法
const signer = await provider.getSigner();
```

### 8.2 问题2：用户信息缓存

**问题描述**：
用户切换账号后，前端仍显示旧用户信息。

**解决方案**：
在登录成功后立即更新WalletContext中的userInfo：
```javascript
const userInfo = await getUserInfo();
setUserInfo(userInfo);
```

### 8.3 问题3：钱包地址不匹配

**问题描述**：
用户可以使用其他人的钱包进行交易。

**解决方案**：
在所有交易前添加钱包地址匹配检查：
```javascript
if (account.toLowerCase() !== userInfo.primaryWalletAddress.toLowerCase()) {
  setError('钱包地址不匹配！');
  return;
}
```
